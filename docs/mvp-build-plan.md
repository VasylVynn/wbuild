# Вітрина — MVP Build-план (v1)

> **Мета:** живий згенерований сайт на власному домені `*.wizz-app.net`, зібраний через розмову з AI-агентом, який власник може відредагувати дотиком і опублікувати — **включно з воронкою заявок у Telegram**.
> **Скоуп:** архітектурний бриф §15 (MVP-скоуп v1, оновлено 2026-07-07). Рішення й «чому» — у брифі; тут — **порядок робіт**.
> **Стек:** Next.js (App Router, TS) на Vercel · Supabase EU (Postgres) · піддомени `*.wizz-app.net` · Sonnet 4.6.
> **Режим:** ship-fast. Кожна фаза закінчується видимою «перемогою», яку можна показати.
> **Статус (2026-07-07):** ✅ Фази 0–4 shipped.

## Легенда
- 🏁 **Перемога фази** — те, що видно на екрані наприкінці.
- ⛓️ **Залежність** — що має бути готове раніше.
- 📄 — посилання на розділ брифу.

*Реальність:* міграції `tenants/pages/conversations/tenant_members/leads` + `app/s/[host]/[[...slug]]` (папки з `_` приватні в Next → перейменовано, журнал #40); `draft_content/published_content` на рівні сторінки; ISR з тег-інвалідацією; 404 для `status='suspended'` (журнал #34).

🏁 Мультитенант-рендер + ISR + kill-switch живуть.

## Фаза 2 — Field-схема вертикалі + генерація ✅ (shipped)
Ціль: з набору фактів AI збирає валідні блоки. Петля генерації жива.

*Реальність:* мультивертикальний реєстр (`florist/bakery/lawyer/autoservice/generic`) — vertical=data; `businessFactsSchema` generic + per-vertical override (labels, price plausibility, theme presets, advisor guidance). Генератор — tool use (structured outputs strict mode відхилив 10-варіантний union, журнал #39). `lead_form` force-injected у кожен сайт кодом (§5.6, журнал #33). Два пресети є; switchTemplate = «Перегенерувати» (журнал #36).

🏁 Факти → валідний сайт з `lead_form`. Генерація детермінована й безпечна.

## Фаза 3 — Агент-чат онбординг ✅ (shipped)
Ціль: нетехнічна людина в розмові доводить сайт до генерації.

*Реальність:* auth відкладено (дашборд = obscurity, журнал #34). Агент = ADVISOR + validator: класифікує вертикаль на 1-му turn, валідатори в коді (телефон/місто/ціна → issues, max 1 питання per turn), ready-gate = рішення моделі. Персистентність shipped: conversations per session, placeholder tenant з `host=null` (O1 вирішено в практиці), resume через localStorage, очищення по finalize (журнал #37). Крок підтвердження = редагована форма з фактами.

🏁 Нова людина → балакає з агентом → отримує сайт з `lead_form` на піддомені.

## Фаза 4 — Редагування + публікація ✅ (shipped)
Ціль: власник сам правит контент і публікує.

*Реальність:* bottom-sheet форма генерується з Zod-схеми (`lib/blocks/fields.ts`, one-registry інваріант); reorder/hide блоків; фото upload: canvas re-encode (EXIF-strip, 1600px, WebP) → `/api/upload` → Supabase Storage; theme preset switcher; «Перегенерувати» = ре-запуск генерації з фактів, старі блоки → pocket (адаптація §4.7). Draft → «Опублікувати» → purge тегів. Дашборд: список заявок (`leads`) + Telegram connect UI (deep-link token flow). 📄 §3, §4.7, §4.8, §5.5, §5.6.

🏁 Повна петля: розмова → сайт → правки дотиком → фото → публікація → заявки в Telegram.

## Поточний цикл (2026-07-08+, після валідації з партнером)
> Джерело: транскрипт обговорення 2026-07-08 + вердикти. Все — без зовнішніх залежностей.

1. **«Відредагувати з ШІ» на рівні блоку** — кнопка в bottom-sheet: AI правит лише пропси блоку (Zod-валідація, тільки draft). Далі — загальний контекстно-свідомий чат редактора.
2. **Візуальна різноманітність** — нові тема-пресети + «скіни» блоків (варіанти верстки) + пікер із прев'ю. **Template-mining**: вподобані безкоштовні шаблони конвертуються (AI, dev-time) у скіни наших блоків — реєстр наповнюється, не ламається (~0.5–1 день/шаблон).
3. **Мобільний аудит** дашборда/редактора (адмінка «з приколами» на телефоні — визнаний борг).
4. **Лендінг-меседж**: продаємо воронку («Клієнти з інтернету — прямо у ваш Telegram»), не «AI-сайт».

## Етап 2 (після MVP) — ріст і монетизація
> Ризик #32 закрито: воронка shipped разом з MVP.

1. **Білінг (UAH)** `💡 2026-07-08, журнал #42`: **WayForPay основний** (регулярні платежі тижд/міс/рік, статуси, підписні сторінки, API) / **LiqPay альтернатива** (Підписка API + нативна розстрочка «Оплата частинами» 2–25). **Stripe відхилено для UA-старту** (нема UA-онбордингу; закордонна юрособа = КІК/бухгалтерія/FX; без UA-розстрочки) — другий рейл для міжнародки через **`BillingProvider`-абстракцію** в коді. Власна state-machine `активний→прострочка→грейс→призупинено` (~3–5 днів; kill-switch уже є). **Код — після ручного тесту готовності платити.** Тарифна механіка на вирішення тоді ж: teaser-заявки («телефон зірочками» понад ліміт) vs чесний ліміт із лічильником — обидві в беклозі.
2. **GBP-інтеграція** (§10.3) — NAP-консистентність, посилання сайту у GBP-лістингу; заявку на GBP API подати рано (розгляд тижнями).
3. **Звіт у Telegram = двигун ретеншну І апселів** — «X переглядів, Y заявок» + AI-рекомендації **з лінком назад у бота** (апсели живуть усередині звіту: мультисторінки, кастом). Лічильники подій почати писати раніше.
4. **Booking-request** (бажана година/дата як опційне поле lead_form) — конфіг полів на вертикаль = дані в реєстрі.
5. **Платні канали заявок**: Viber-пуш і SMS (TurboSMS) — add-on'и (реальна собівартість за повідомлення); Telegram лишається безкоштовним (собівартість ~0).
6. **«Оновлюй мене сам»** — місячний пінг від бота: власник кидає фото/новини → AI оновлює draft.
7. **Кастом-замовлення** — кнопка «Хочу кастомні зміни» → заявка нам → рахунок (~$300); concierge-клапан як апсел + генератор лідів.
8. **Мультисторінки** (сторінка на послугу — модель даних готова, §5.1.1) — апсел, тригериться рекомендацією у звіті.
9. **Фото/лого з вивіски** — «сфотографуй вивіску» як шлях лого (аудиторія не вантажить SVG); image-AI (~4 грн/фото) — експеримент із бюджетом, межі §4.8.
10. Власні домени (Задача 2), реєстрація .com.ua (Задача 3) — при платному попиті; націнка $2–3 — потім, на пролонгації.
