# Конкурент RoxyBo — повний розбір і напрям руху

> **Дата:** 2026-07-11. Джерела: HAR (903 запити) залогіненої сесії + HTML їхнього редактора + згенерований сайт FLORA ART + скріншоти. Аналіз: 5 паралельних агентів (фічі/UX, дизайн-розрив, архітектура, чесний критик, ринок) + адверсарна валідація рекомендацій (синтез Fable).
> **Мандат власника:** можемо міняти все, крім ядра ідеї (сайт + заявки для локального бізнесу). Документ — без води, з конкретикою.

## 0. TL;DR (якщо читати лише це)

1. **Дизайн-розрив «у них гарніше» — переважно наші дешеві баги, не їхня магія.** Наші тема-пресети обіцяють шрифти Playfair/Nunito, а tenant-сайти їх **взагалі не вантажать** (тихий фолбек на системні). Нуль hover-станів, нуль анімацій появи. Це полірувальний шар на ~1 день, не переробка продукту.
2. **Архітектуру НЕ переписувати.** Їхній код-ген на клієнта (окремий Flask+SQLite на сайт) — пастка для тонкомаржинального продукту. Наш шар шаблонів (studio) уже доводить: їхній рівень краси виразимий в одному кодбейсі з 95% маржею.
3. **Головна правда:** ми будуємо платформу для масштабу (адмінка, rate-limits, дизайн-паки) **маючи нуль платних клієнтів**, поки RoxyBo вже монетизований. Найбільший ризик — не RoxyBo, а те, що ми досі не запустили банерний тест центральної гіпотези.
4. **Клин загострити, не міняти:** їхня широта (сайти+ігри+боти) = наша перевага проти них. Для 45+ власника універсальний генератор виглядає страшніше за інструмент «рівно під один зрозумілий сценарій — заявки в Telegram».

## 1. Що таке RoxyBo — стек і архітектура

**Платформа** (`roxybo.com.ua`): nginx + **PHP** (PHPSESSID) + jQuery + Alpine.js + Tailwind CDN (редактор). Аналітика: **PostHog + Facebook Pixel + Google Tag Manager** (серйозна воронкова дисципліна). Авторизація: Google OAuth + email. Інфра: AWS ECS + S3. Генерація **асинхронна через polling** (`get_dialog` → `{current_step, messages, thinking}`), є **thinking-режим**.

**Згенеровані сайти** (напр. `flower-shop-vizitka-1648.roxybo.com.ua`): **окремий Flask + Jinja + SQLite застосунок на КОЖНОГО клієнта**. Багатосторінкові (`/`, `/order`, `/gallery`, `/contacts`, `/admin/*`), з **per-site адмін-панеллю CRUD** (dashboard/сторінки/галерея/послуги/заявки/юзери). Дизайн: Tailwind CDN + custom CSS з анімаціями + Google-serif. Фото — AI-генеровані.

**Скоуп:** сайти + **ігри (Tetris) + боти + застосунки**; домени (мультиреєстратор, живі ціни UAH); **експорт коду**; оплата; SEO. Тарифи: Промо 0₴ → Стандарт 649₴ → Бізнес 1549₴ → Преміум 4149₴/міс, **токенна модель** + овередж за токен.

Позиціонування: «Від ідеї до готового сайту за 4 хвилини». Горизонтальний генератор, **не воронка-first**.

## 2. Що вони роблять КЛАС (кандидати на копіювання)

### Дизайн (чому їхні сайти виглядають преміально — конкретно)
- **Парне шрифтування**: `display: Playfair Display` + `body: Montserrat`; лого `tracking-widest`, h1 до `text-8xl`.
- **Ступінчаста поява**: `animate-fade-in-up` з `animation-delay: 0.2s/0.4s/0.6s` (бейдж→h1→CTA); scroll-reveal на секціях.
- **Eyebrow як pill**: `px-4 py-1 border border-white/40 rounded-full uppercase tracking-widest`.
- **Hover-фізика кнопок**: `hover:scale-105`, скляна друга кнопка `bg-white/10 backdrop-blur-md`.
- **Декор без фото**: blur-blob `bg-accent/20 blur-3xl`, гігантський напівпрозорий гліф-watermark, noise-текстура, курсив на слові в заголовку.

### UX
- **Block picker** — клік по елементу сайту в прев'ю замість опису словами → точне таргетування для AI (топ-фіча).
- **Автовиявлення + самовиправлення JS-помилок** в iframe без участі юзера.
- **Гостьовий чат до реєстрації** — value видно перед сайн-апом.
- **Thinking-індикатор у чаті** («Думаю…»).
- **Device-preview** десктоп/планшет/мобільний.
- **Автобекапи/версії** сайту — safety-net від AI-помилок.
- **Завжди видимий баланс** токенів/₴ у composer, live-оновлення, блокування при 0.

### Продукт
- Мультисторінковість, per-site адмінка, домени з живими цінами, прозорий прайс на екрані.

## 3. Що вони роблять ПОГАНО / не для нас

- **Код-ген на клієнта = ops/security-борг.** N Flask+SQLite = N хостів/патчів/атак-поверхонь; одне CVE = тисячі редеплоїв. Видимі security-smell: сирий SQL конкатенацією, `password_hash='some_strong_password_hash_here'`, дефолтний `admin@example.com`, кука `__http_status`. Нульова маржа проти нашої 95%.
- **Вигадані фото приміщень.** Авто-СТО отримав згенерований «промисловий гараж» — відвідувач приймає за реальний. Для юриста/лікаря/СТО це межує з введенням в оману (наш інваріант #44 це забороняє).
- **Токенна модель** — когнітивне навантаження + ризик сюрприз-рахунку; антитеза «радикальній простоті» для 45+.
- **Voice-input із багом**: `recognition.lang` захардкожено `'ru-RU'` навіть на укр-сайті.
- **Тихий авто-апгрейд** з промо-тарифу через 30 днів — довіра-ризик.
- **Штучний stagger** статус-повідомлень — косметичний трюк perceived-performance.
- **Горизонтальність** (ігри/боти/застосунки) — виглядає страшніше й складніше для нашої ЦА.

## 5. Що в нас реально краще (не cope)

- **Воронка заявок у Telegram force-injected** у кожен сайт — у RoxyBo цього як ядра взагалі нема.
- **Чесність зображень** (#44) — вони фабрикують гаражі, ми технічно неспроможні на цю помилку.
- **Мультитенантність**: один застосунок / N рядків БД — собівартість, швидкість фіксу (push раз = усі сайти), надійність. Це рів, не обмеження.
- **Grounding-валідація** фактів (телефон/ціна/адреса 1:1) + block-registry проти галюцинацій структури — суворіше за все видиме в їхньому API.
- **Редактор дотик-поле** (накосячити неможливо) — для 45+ кращий за їхню CRUD-панель.
- **Шар шаблонів (studio)** уже доводить, що їхній рівень краси виразимий у нас без переписування.

## 6. Архітектурний вердикт (Opus)

Data-driven мультитенантність — **НЕ стеля**. `PageRenderer` уже роздвоює рендер: якщо резолвиться `templateId` — блок малюється кастомним React-компонентом шаблону всередині його `wrapper` (власний фон/шрифти/анімації, може бути темним), а не дефолтним скіном. Тобто схема блоків обмежує **форму контенту** (які поля існують), не візуальне багатство. Multi-page вже вбудована в модель даних (`pages(tenant_id, slug)`, резолвер `[[...slug]]`).

Реальні межі — **словник блоків** (каталог із варіаціями/бронювання не виразити наявними 11 схемами) і **людська праця над бібліотекою шаблонів** (лінійна за к-стю шаблонів×вертикалей). Обидві — керовані витрати, не архітектурні стіни. Їхній код-ген — протилежний обмін: вища стеля per-site ціною N-застосунків боргу й нульової маржі. **Не переписувати; вкладати в шаблони + multi-page, а не в rebuild.**

## 7. Рекомендації з валідацією (вердикт Fable-адверсара)

- **R1. Полагодити шрифти tenant-сайтів** (`next/font` у shell). ✅ **Робити першим.** Найбільший impact/зусилля, каскадом на 4+ пресети. Зламаний контракт «назва пресету обіцяє шрифт».
- **R2. Дизайн-полірувальний шар** (hover-стани, staggered fade-in-up, eyebrow-pill, ширша типошкала, watermark/blob/noise для секцій без фото — в межах #44). ✅ **Робити,** але **не золотити**: рівно щоб вивід був конкурентним для банера, не більше.
- **R3. Дешеві UX з RoxyBo.** ⚠️ **Розділити:** thinking-індикатор у чаті + device-preview тумблер — ✅ дешево, беремо (device-preview і так у десктоп-поліровці). Повна історія версій/автобекапи — ⚠️ **відкласти**: у нас уже є pocket від «Перегенерувати»; достатньо винести **одну кнопку «Скасувати останню зміну»** поверх pocket, а повну систему бекапів не будувати до валідації.
- **R4. Multi-page генерація.** ⚠️ **НЕ будувати зараз.** Модель готова, але для search-intent вертикалей (СТО/юрист/майстри) one-page достатньо і не косить конверсію. Вмикати лише коли конкретна каталог-вертикаль (пекарня/квіткарня) цього вимагатиме після валідації. Зараз = scope creep.
- **R5. Контр-позиція проти токенів + сторінка цін.** ⚠️ Сторінка «одна ціна, без лічильників і сюрприз-рахунків» — ✅ дешево й загострює клин; але **ціну тримати як гіпотезу** (ручний тест готовності платити реквізитами лишається гейтом білінг-коду). Не анкорити ринок передчасно.
- **R6. НЕ гнатися** за іграми/ботами/застосунками, експортом коду, per-client codegen, per-site CRUD-адмінкою. ✅ **Категорично.** Експорт коду = продаж слабкості їхньої архітектури як фічі + непотрібний 45+ аудиторії.
- **R7. ЗАМОРОЗИТИ подальше будівництво платформи, запустити банерний + друзі-тест ЗАРАЗ.** ✅ **Це головний виклик.** R1/R2 — єдина виправдана до-валідаційна робота (роблять банерний тест чесним щодо якості виводу). Усе інше — після даних.
- **R8. Загострити клин:** «інструмент, що робить рівно одне зрозуміле — сайт локального бізнесу, чиї заявки падають у твій Telegram; одна ціна; справжні фото». ✅ Проти широти RoxyBo це працює тим сильніше, що ширшим стає конкурент.

### Сліпі зони, які адверсар додав би
1. **Банерний тест — це і є гейт; усе інше вторинне.** Ми знову будуємо замість запустити експеримент, що відповідає на центральну гіпотезу («чи конвертує self-serve для 40+»).
2. **Монетизація RoxyBo ≠ доказ, що НАША ніша платить.** Їхні платники можуть бути іншим сегментом (ті, кому треба «будь-що», не 45+ квіткарки). Не переоцінювати «ринок доведено».
3. **Дизайн тепер — table-stakes, не диференціатор.** Треба дотягнути до паритету, щоб не втратити клік; але виграють воронка + довіра + простота, не типографіка.

### Найнебезпечніша рекомендація
Сприйняти весь цей розбір як мандат **будувати більше** (multi-page, бекапи, паритет адмінки) замість **полірувати-і-валідувати**. Саме це годує пастку, яку назвав критик: платформа для масштабу з нулем платних клієнтів.

## 8. Напрям руху (пріоритезовано)

**Тиждень 1 — полірування виводу (єдина виправдана до-валідаційна робота):**
1. R1 шрифти tenant-сайтів (½ дня).
2. R2 дизайн-шар: hover, staggered-поява, eyebrow-pill, декор без фото (1–1.5 дня).
3. Device-preview тумблер + thinking у чаті + кнопка «Скасувати останню зміну» поверх pocket (½ дня).

**Далі — ВАЛІДАЦІЯ (не код):**
5. Банерний A/B (2 меседжі: «заявки в Telegram» vs «щоб знаходив Google») + партнер особисто дзвонить + друзі-тест (§6 партнерської розмови). Ручний тест готовності платити реквізитами.

**Після даних — лише те, що попит підтвердить:**
6. Сторінка цін, домени, multi-page для каталог-вертикалей, білінг — кожне тільки коли валідація покаже потребу.

**Ніколи:** per-client codegen, ігри/боти, експорт коду, токенна модель.
