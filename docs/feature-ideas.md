# Вітрина — ідеї фіч та оцінка реалізовності

*Робочий документ. Позиціонування: не "AI робить сайт", а "клієнти з інтернету для власників бізнесу 50+, прямо у месенджер". Сайт — лише механізм.*

---

## 1. Ядро продукту (вже узгоджено)

### 1.1 Адаптивний AI-інтерв'юер
Чат-бот ставить питання залежно від типу бізнесу (юристу — про практики та консультації, перукарні — про послуги і запис). Це головна диференціація проти Wix ADI / Weblium, які питають шаблонно.
- **Складність:** низька. Це промпт-інжиніринг + структура даних по нішах. Жодних зовнішніх API.
- **Статус:** ядро MVP.

### 1.2 Редагування без drag-and-drop
Власник бачить прості поля по секціях: "перегенерувати текст", "змінити вручну", "замінити фото". Жодного візуального конструктора.
- **Складність:** середня. Потрібна чиста структура контенту (JSON-схема секцій), але це повністю під нашим контролем.
- **Статус:** ядро MVP.

### 1.3 Лід-форма на сайті → повідомлення власнику
Відвідувач заповнює форму, власник миттєво отримує повідомлення.
- **Складність:** залежить від каналу доставки (див. розділ 3).
- **Статус:** ядро MVP. Це те, що перетворює сайт на джерело клієнтів.

## 2. Фічі для утримання підписки (щомісячна цінність)

### 2.1 Щомісячний звіт простою мовою
"Ваш сайт відвідали 214 людей, 12 заповнили форму, 8 подзвонили. Найбільше — з Google за запитом 'адвокат Львів'." Три речення, не Google Analytics.
- **Складність:** низько-середня. Власна аналітика (page views, кліки на телефон, submit форми) — простий власний трекер або Plausible/self-hosted. Пошукові запити — з Google Search Console API (безкоштовний, стабільний, добре документований).
- **Ризик:** мінімальний. Це найдешевша фіча з найбільшим ефектом на retention.

### 2.2 AI-відповідач на заявки
Заявка о 23:00 → AI одразу відповідає відвідувачу від імені бізнесу ("Дякуємо, Олег Іванович зв'яжеться зранку..."), власнику приходить кваліфікований лід.
- **Складність:** низька технічно (email/SMS відвідувачу + LLM-відповідь), середня продуктово (треба guardrails, щоб AI не наобіцяв зайвого — ціни, гарантії).
- **Статус:** фаза 2, після MVP.

### 2.3 Запис на візит (прості послуги)
Простий календар для перукарень, стоматологів, майстрів. Без складності Calendly.
- **Складність:** середня. Календар + слоти + нагадування. Можна почати з "запит на запис" (форма з бажаним часом), а не реальним календарем — це знімає 80% складності.
- **Статус:** фаза 2, тільки для ніш послуг.

### 2.4 "Оновлюй мене сам"-режим
Раз на місяць бот питає власника: "Є новини? Нові фото? Акція?" — власник кидає фото і два речення, AI оновлює сайт.
- **Складність:** низька, якщо канал зв'язку вже працює (розділ 3). Свіжий контент також допомагає SEO.
- **Статус:** фаза 2.

### 2.5 QR-візитки (бонус)
- **Складність:** генерація QR — тривіальна. Друк і доставка — операційний головний біль.
- **Рішення:** давати PDF для самостійного друку, фізичний друк не робити. Відкладено.

## 3. Канал доставки повідомлень — головне відкрите питання

### 3.1 Viber — реально, але з нюансами
- Viber має офіційний **Bot API** (REST, безкоштовний вхід). Ключове обмеження: бот може писати користувачу тільки після того, як **користувач сам ініціював діалог** (натиснув Start / написав боту). Тобто онбординг: власник сканує QR → відкриває бота → тисне Start → ми зберігаємо його ID → далі шлемо повідомлення без обмежень.
- Viber періодично змінює комерційну політику для ботів (сесійна тарифікація для бізнес-повідомлень) — умови треба перевірити на момент реалізації.
- **Альтернатива через агрегаторів** (TurboSMS, BSG, Infobip): Viber Business Messages без власного бота, але платно за повідомлення і потребує реєстрації відправника.
- **Вердикт:** автоматизувати можна. Складність середня, не висока.

### 3.2 Telegram — найпростіший запасний план
- Bot API повністю безкоштовний, без модерації, документація найкраща на ринку. Той самий флоу (користувач тисне Start).
- Серед української аудиторії 50+ Viber досі популярніший, але Telegram-покриття зростає.
- **Рекомендація:** будувати абстракцію "канал повідомлень" і запускати **Telegram першим** (день роботи), Viber — другим (тиждень+юрособа для реєстрації бота). SMS через TurboSMS — як fallback для тих, хто не в месенджерах.

### 3.3 Практичний висновок
Не блокувати MVP Вайбером. MVP: email + Telegram. Viber додається, коли є перші клієнти, які його просять.

## 4. Google Business Profile — чесна оцінка

- **GBP API існує** (Business Profile APIs), дозволяє створювати та керувати картками. АЛЕ: доступ до API треба **запитувати у Google** (форма, розгляд заявки, зазвичай дають агенціям/платформам — наш кейс якраз підходить, Wix і Weblium так працюють).
- **Верифікацію бізнесу автоматизувати не можна** — Google вимагає підтвердження власником (відео-верифікація, листівка, телефон). Це завжди ручний крок власника.
- **Реалістична модель:**
  - *MVP:* напівручний "concierge" — покроковий гайд + ми готуємо всі тексти/фото/категорії, власник тільки проходить верифікацію за нашою інструкцією (або по телефону з нами).
  - *Пізніше:* після отримання API-доступу — автоматичне створення картки, синхронізація даних сайт↔GBP, публікація постів, відповіді на відгуки через AI.
- **Вердикт:** повна автоматизація на старті — ні. Concierge-версія — так, і для перших 50 клієнтів цього достатньо. Це навіть плюс: особистий контакт з ЦА 50+ будує довіру.

## 5. AI-провайдер (Anthropic/Claude)

- Технічних блокерів немає: генерація текстів, адаптивне інтерв'ю, перегенерація секцій — стандартні API-виклики. Claude добре працює українською.
- **Що врахувати:**
  - Вартість: генерація одного сайту ≈ десятки тисяч токенів — це копійки. Основні витрати підуть на AI-відповідач і "оновлюй сам", але вони теж дешеві на Haiku/Sonnet.
  - Закласти абстракцію над LLM-провайдером (Claude / Gemini / OpenAI взаємозамінні) — щоб не залежати від цін одного вендора.
  - Guardrails для AI-відповідача: заборона називати ціни/гарантії без підтвердження власника.
- **Вердикт:** найменш ризикована частина всієї системи.

---

## 6. Чого свідомо НЕ робимо

- ❌ E-commerce / кошик / оплата на сайті — інший продукт, прірва складності.
- ❌ Інтеграції з CRM — у ЦА немає CRM, месенджер і є їхня CRM.
- ❌ Багатосторінкові блоги.
- ❌ Drag-and-drop редактор.
- ❌ Фізичний друк візиток.

---

## 7. Пріоритети (пропозиція)

| Фаза | Що входить |
|------|-----------|
| **MVP (timebox 4–6 тижнів)** | Адаптивний інтерв'юер → генерація сайту → публікація; лід-форма → email + Telegram; редагування через прості поля |
| **Фаза 2** | Viber-канал; щомісячний звіт; GBP concierge; AI-відповідач |
| **Фаза 3** | GBP API-автоматизація; запис на візит; "оновлюй мене сам"; батли за наступні країни 🙂 |

**Критерій успіху MVP:** нетехнічна людина за 15 хвилин діалогу отримує опублікований пристойний сайт без твоєї допомоги.
